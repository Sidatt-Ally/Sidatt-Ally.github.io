Cyclistic Bike-Share — Rapport d'Analyse

Comment convertir les cyclistes occasionnels en membres annuels ?

Analyste Sidi Mohamed ALLY
Certification Google Data Analytics
Période Janvier 2019 – Décembre 2020
Date Juin 2026
01 — Contexte business

Cyclistic est un service de vélo en libre-service basé à Chicago avec 5 824 vélos répartis sur 692 stations. Le service propose deux types d'utilisateurs :

Membres
Abonnement annuel — plus rentables
Casual
Ticket jour / trajet unique
Question analytique

Comment les membres annuels et les cyclistes occasionnels utilisent-ils les vélos Cyclistic différemment ? L'objectif est d'identifier les leviers marketing pour convertir les casual en membres.

04 — Résultats de l'analyse
Résultat 1 — Volume & Durée

Les membres génèrent 59% des trajets mais roulent 1,48x moins longtemps que les casual. Les casual privilégient des trajets longs et détendus (loisir), les membres des trajets courts et efficaces (domicile-travail).

Résultat 2 — Comportement par jour de la semaine

Les membres ont leur pic le jeudi — profil de navetteur. Les casual explosent le samedi — profil de loisir week-end. La durée moyenne casual le samedi est 1,6x celle des membres.

Fig. 3 — Nombre de trajets par jour de la semaine
Fig. 4 — Durée moyenne par jour
Résultat 3 — Saisonnalité

52% des trajets casual se concentrent en été (juin–août). Les membres restent actifs toute l'année (556k trajets en hiver vs 57k pour les casual). Fenêtre de conversion optimale : mai–août.

Fig. 5 — Evolution mensuelle 2019–2020
Résultat 4 — Heure de départ

Membres : double pic à 8h et 17h (heures de rush) → navetteurs confirmés. Casual : montée progressive de 6h à 15h, aucun pic rush → loisir et tourisme.

Fig. 6 — Répartition par heure de départ
Résultat 5 — Zones géographiques (casual)

Les casual se concentrent sur les stations touristiques : Lakefront Trail, Millennium Park, Navy Pier, Shedd Aquarium. Ce sont les zones cibles pour la campagne marketing.

Fig. 7 — Top 10 stations de départ — Casual riders
Résultat 6 — Types de vélos

Les casual utilisent proportionnellement plus d'electric bikes (15,2% vs 11,8% pour les membres). Les docked bikes dominent les deux groupes.

Fig. 8 — Types de vélos utilisés par groupe
05 — Recommandations
01
Campagne estivale de conversion
Lancer en mai, cibler juin–août. Offre -20% sur l'abonnement annuel déclenchée après le 3e trajet du mois. Message : « Vous aimez Cyclistic — passez à l'illimité ».
02
Marketing géolocalisé week-end
Cibler les stations touristiques le samedi matin (Millennium Park, Navy Pier). Affiches + QR code vers page d'abonnement. Partenariat offices de tourisme de Chicago.
03
Pass week-end vers abonnement annuel
Créer un pass samedi+dimanche à prix attractif. Offre de bascule vers l'annuel après 3 week-ends. Tracker la conversion et ajuster après 6 mois.

SQL   Statistiques descriptives par groupe ▼ Afficher 
SELECT
    member_casual,
    COUNT(*)                                                    AS total_rides,
    ROUND(AVG(ride_length), 2)                                  AS avg_ride_length_min,
    ROUND(PERCENTILE_CONT(0.5) WITHIN GROUP
          (ORDER BY ride_length), 2)                            AS median_ride_length_min,
    ROUND(MAX(ride_length), 2)                                  AS max_ride_length_min,
    ROUND(STDDEV(ride_length), 2)                               AS std_ride_length_min
FROM cyclistic_clean
GROUP BY member_casual
ORDER BY member_casual;
SQL   Trajets par jour de la semaine ▼ Afficher 
SELECT
    member_casual,
    CASE day_of_week
        WHEN 1 THEN 'Dimanche' WHEN 2 THEN 'Lundi'
        WHEN 3 THEN 'Mardi'    WHEN 4 THEN 'Mercredi'
        WHEN 5 THEN 'Jeudi'    WHEN 6 THEN 'Vendredi'
        WHEN 7 THEN 'Samedi'
    END                             AS day_name,
    COUNT(*)                        AS total_rides,
    ROUND(AVG(ride_length), 2)      AS avg_ride_length_min
FROM cyclistic_clean
GROUP BY member_casual, day_of_week
ORDER BY member_casual, day_of_week;
SQL   Saisonnalite avec WINDOW FUNCTION ▼ Afficher 
SELECT
    member_casual,
    season,
    COUNT(*)                        AS total_rides,
    ROUND(AVG(ride_length), 2)      AS avg_ride_length_min,
    ROUND(COUNT(*) * 100.0 /
          SUM(COUNT(*)) OVER (PARTITION BY member_casual), 1)
                                    AS pct_of_user_type
FROM cyclistic_clean
GROUP BY member_casual, season
ORDER BY member_casual,
    CASE season
        WHEN 'Spring' THEN 1 WHEN 'Summer' THEN 2
        WHEN 'Fall'   THEN 3 WHEN 'Winter' THEN 4
    END;
SQL   Top 10 stations casual (cibles marketing) ▼ Afficher 
SELECT
    start_station_name,
    COUNT(*)                        AS total_rides
FROM cyclistic_clean
WHERE member_casual = 'casual'
  AND start_station_name IS NOT NULL
GROUP BY start_station_name
ORDER BY total_rides DESC
LIMIT 10;
SQL   CTE + ROW_NUMBER — Jour de pointe par groupe ▼ Afficher 
WITH ranked_days AS (
    SELECT
        member_casual,
        day_of_week,
        COUNT(*)                AS rides,
        ROW_NUMBER() OVER (
            PARTITION BY member_casual
            ORDER BY COUNT(*) DESC
        )                       AS rnk
    FROM cyclistic_clean
    GROUP BY member_casual, day_of_week
)
SELECT
    member_casual,
    CASE day_of_week
        WHEN 1 THEN 'Dimanche' WHEN 2 THEN 'Lundi'
        WHEN 3 THEN 'Mardi'    WHEN 4 THEN 'Mercredi'
        WHEN 5 THEN 'Jeudi'    WHEN 6 THEN 'Vendredi'
        WHEN 7 THEN 'Samedi'
    END                         AS peak_day,
    rides
FROM ranked_days
WHERE rnk = 1
ORDER BY member_casual;

Python   Statistiques descriptives avec pandas ▼ Afficher 
import pandas as pd

df = pd.read_csv('cyclistic_clean.csv', low_memory=False)

desc = df.groupby('member_casual')['ride_length'].agg(
    total_rides  = 'count',
    mean_min     = 'mean',
    median_min   = 'median',
    max_min      = 'max',
    std_min      = 'std'
).round(2).reset_index()

# Ajouter le pourcentage du total
desc['pct'] = (desc['total_rides'] / desc['total_rides'].sum() * 100).round(1)
print(desc)
Python   Analyse par jour de la semaine ▼ Afficher 
DOW_LABELS = {1:'Dim', 2:'Lun', 3:'Mar', 4:'Mer',
              5:'Jeu', 6:'Ven', 7:'Sam'}

dow = df.groupby(['member_casual', 'day_of_week']).agg(
    total_rides  = ('ride_id',     'count'),
    avg_length   = ('ride_length', 'mean')
).round(2).reset_index()

dow['day_name'] = dow['day_of_week'].map(DOW_LABELS)

# Pourcentage par type d'utilisateur
dow['pct'] = dow.groupby('member_casual')['total_rides'].transform(
    lambda x: (x / x.sum() * 100).round(1)
)

import matplotlib.pyplot as plt
import matplotlib.ticker as mticker

PALETTE = {'member': '#1A73E8', 'casual': '#E84D0E'}

    df[(df['member_casual'] == 'casual') &
       (df['start_station_name'].notna())]

Python   Connexion PostgreSQL + analyse depuis la base ▼ Afficher 
import psycopg2
import pandas as pd

conn = psycopg2.connect(
    host='localhost', port=5432,
    dbname='cyclistic_db',
    user='postgres', password='****'
)

query = """
    SELECT member_casual, season,
           COUNT(*) AS total_rides,
           ROUND(AVG(ride_length), 2) AS avg_min,
           ROUND(COUNT(*) * 100.0 /
                 SUM(COUNT(*)) OVER (PARTITION BY member_casual), 1) AS pct
    FROM cyclistic_clean
    GROUP BY member_casual, season
    ORDER BY member_casual, pct DESC;
"""

df = pd.read_sql(query, conn)
conn.close()
print(df.pivot(index='season', columns='member_casual', values='pct'))
